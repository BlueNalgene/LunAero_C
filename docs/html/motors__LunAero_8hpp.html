◆ motor_handler()

void motor_handler ( )

This function handles the motor commands by running in a loop and checking the value from the motor struct. Stopping the motors is handled by STOP_DIR values, and gradually slows the motor down to prevent abrupt stops shaking the video too much. Next, the function handles vertical and horizontal motor commands sequentially. If the motor command was a positive value and the same as the command in the previous cycle, the function calls speed_up. In the case of horizontal motion, if the motor direction abruptly switches, the loose wheel protocol is summoned which forces high speed motion for a number of seconds based on the value LOOSE_WHEEL_DURATION in settings.cfg.

◆ speed_up()

void speed_up ( int  motor)

This function increases the speed of the motor's movement by setting appropriate PWM values. The speed never drops below a minimum value (full stops ignore this function). Speed is incremented by one percent PWM to a maximum value.

Parameters
motorThe motor to run speed settings on. 1 = vertical, 2 = horizontal
